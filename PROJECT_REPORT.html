Project Overview

Objective: Build an AI-powered browser game that uses a webcam and a custom-trained machine learning model to recognize Rock, Paper, and Scissors hand gestures in real time — then play a full game against a computer opponent, complete with scoring, win streaks, and voice announcements.

The application is a full-stack frontend project built from scratch using React 18 and Vite 4. It integrates Google Teachable Machine (an ML model trained by the developer) and TensorFlow.js loaded via CDN. A custom gesture-recognition hook reads the webcam feed frame-by-frame, classifies the user's hand pose at the right moment, and a game engine determines the winner. The UI supports light/dark themes, real-time streak tracking, and Text-to-Speech announcements.

Phase 1 — Training the AI Model with Google Teachable Machine

Before writing a single line of application code, the underlying machine learning model was trained using Google Teachable Machine — a free, browser-based tool by Google that lets anyone train image classification models without writing ML code.


Why Teachable Machine? Training a gesture-recognition model from scratch with raw TensorFlow would require thousands of lines of Python, a GPU, and days of tuning. Teachable Machine abstracts this entirely — training takes under 10 minutes in the browser using your own webcam samples.


1

Open Google Teachable Machine

Navigate to teachablemachine.withgoogle.com and click "Get Started", then choose "Image Project" → "Standard image model". This opens the training interface with an empty project.

2

Create 4 Classes

The model was trained on exactly 4 classes:

  • Rock — closed fist ✊
  • Paper — open flat hand ✋
  • Scissors — two-finger V sign ✌️
  • No Gesture — background / neutral / no hand visible

The "No Gesture" class is critical — without it, the model would always force-predict one of the game gestures even when there is no hand in frame, causing false positives.

3

Capture Training Samples via Webcam

For each class, the webcam was used to record samples by holding the gesture clearly in front of the camera. Approximately 200–300 image samples per class were captured across:

  • Different angles (straight on, slight tilt left/right)
  • Different lighting conditions (bright, dim)
  • Different distances from camera (close, medium)
  • Both hands (where applicable)

More diverse samples = more robust real-world detection.

4

Train the Model

Click "Train Model". Teachable Machine uses transfer learning on top of MobileNet (a lightweight CNN pre-trained on millions of images by Google). Training fine-tunes only the final classification layer on the 4 custom gesture classes. In the browser, training typically completes in 2–5 minutes on a standard laptop, even without a GPU.

  • Default: 50 epochs, batch size 16, learning rate 0.001
  • The training chart shows accuracy rising toward 100% as epochs progress
5

Test the Model Live

Teachable Machine provides a real-time preview pane. Each gesture was tested live — a confidence bar shows the probability assigned to each class. The model should show > 90% confidence on the correct class when the gesture is held clearly.

6

Export / Publish the Model

Click "Export Model" → "Tensorflow.js" → "Upload (shareable link)". Teachable Machine uploads the model files to Google's servers and returns a unique public URL. This was then saved as the MODEL_URL constant in the application:

// src/utils/constants.js
export const MODEL_URL = 'https://teachablemachine.withgoogle.com/models/JfhfXGxkFl/';

The model serves two files at that URL path:

  • model.json — model architecture + weights
  • metadata.json — class labels (Rock, Paper, Scissors, No Gesture)

1

Scaffold the React App with Vite

Vite was chosen over Create React App because it starts in milliseconds, has native ESM support, and produces smaller production bundles.

2

Load TensorFlow.js and Teachable Machine via CDN

Instead of installing these as npm packages (which causes peer-dependency conflicts with React 18), both libraries are loaded as CDN <script> tags in index.html:

<!-- index.html — exact version pairing required -->
<script src="https://cdn.jsdelivr.net/npm/@tensorflow/tfjs@1.3.1/dist/tf.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/@teachablemachine/image@0.8/dist/teachablemachine-image.min.js"></script>

This makes window.tmImage available globally before the React app boots.

3

Define All Constants Up Front

All magic values (model URL, gesture names, game states, emoji mappings, TTS messages) were centralized in src/utils/constants.js so they can be imported anywhere without repetition:

export const GESTURES = { ROCK: 'Rock', PAPER: 'Paper', SCISSORS: 'Scissors', NO_GESTURE: 'No Gesture' };
export const GAME_STATES = { IDLE: 'idle', COUNTDOWN: 'countdown', RESULT: 'result' };
export const PREDICTION_THRESHOLD = 0.7; // 70% confidence minimum

Phase 4 — Key Implementation Details

5.1 — Loading the Model & Webcam (useTeachableMachine.js)

On component mount, the hook loads the Teachable Machine model from its hosted URL, then requests webcam access from the browser. Both operations run asynchronously in parallel-safe sequence:

// Load model from Google's CDN using the exported model URL
const tmImage = window.tmImage; // injected via <script> tag in index.html
modelRef.current = await tmImage.load(
  MODEL_URL + 'model.json',
  MODEL_URL + 'metadata.json'
);

// Open webcam stream
const stream = await navigator.mediaDevices.getUserMedia({
  video: { facingMode: 'user', width: 640, height: 480 },
});
video.srcObject = stream;
await video.play();

5.2 — Gesture Prediction with Confidence Threshold

When the countdown hits zero, predictGesture() runs a single inference pass on the current video frame. A 70% confidence threshold filters out uncertain predictions:

const predictions = await modelRef.current.predict(videoRef.current);
const best = predictions.reduce((a, b) => a.probability > b.probability ? a : b);

// Only accept if confidence ≥ 70% AND not "No Gesture"
if (best.probability >= 0.7 && best.className !== GESTURES.NO_GESTURE) {
  return best.className; // "Rock" | "Paper" | "Scissors"
}
return null; // triggers NO_GESTURE action

5.3 — Game State Machine (useGameLogic.js)

All game state is managed by a useReducer — a predictable, testable state machine with three core states:

idle
→
countdown
→
result
→
countdown

idle → START_GAME → countdown → CAPTURE_GESTURE / NO_GESTURE → result → PLAY_AGAIN → countdown

case 'CAPTURE_GESTURE': {
  const { gesture } = action.payload;
  const computerMove = getComputerMove();          // random pick
  const result = determineWinner(gesture, computerMove); // pure function
  const newStreaks = updateStreaks(result, state);   // pure function

  return {
    ...state, gesture, computerMove, result,
    gameState: GAME_STATES.RESULT,
    userScore: result === 'user_win' ? state.userScore + 1 : state.userScore,
    ...newStreaks,
  };
}

5.4 — Game Rules (gameRules.js)

The win logic is a simple lookup table rather than a chain of if/else conditions:

export function determineWinner(userGesture, computerMove) {
  if (userGesture === computerMove) return 'draw';
  const winsAgainst = {
    Rock: 'Scissors',
    Paper: 'Rock',
    Scissors: 'Paper',
  };
  return winsAgainst[userGesture] === computerMove ? 'user_win' : 'computer_win';
}

5.5 — Text-to-Speech (useSpeech.js)

The Web Speech API speechSynthesis announces each event. Previous utterances are cancelled before each new one to prevent queuing:

const speak = useCallback((text) => {
  window.speechSynthesis.cancel();           // stop any ongoing speech
  const utterance = new SpeechSynthesisUtterance(text);
  window.speechSynthesis.speak(utterance);
}, []);

5.6 — Light / Dark Theme

Theme switching is done by toggling a single HTML attribute — all CSS variables update automatically:

/* variables.css — same property, two theme contexts */
:root                    { --color-bg: #ffffff; --color-primary: #2563eb; }
[data-theme="dark"]      { --color-bg: #0f172a; --color-primary: #60a5fa; }

// App.jsx — one line to switch theme
document.documentElement.setAttribute('data-theme', isDark ? 'dark' : 'light');

Complete Game Flow — Step by Step

1

Page Load

TensorFlow.js and Teachable Machine load from CDN. useTeachableMachine fires, loads the model from MODEL_URL and requests webcam permission. Once both are ready, status dots turn green: ● Camera ● AI Model. The Start button activates.

2

User Clicks "Start Game"

START_GAME action dispatched → gameState becomes 'countdown'. The countdown hook starts: 3 → 2 → 1 displayed with a 1-second interval.

3

Countdown Reaches Zero

TTS announces "Show your move!". predictGesture() captures a single frame from the live video and runs inference. The model returns confidence probabilities for all 4 classes.

4a

Gesture Detected (confidence ≥ 70%)

CAPTURE_GESTURE action fires with the detected gesture. The reducer picks a random computer move, determines the winner, updates scores and streaks, transitions to 'result' state. TTS announces the result.

4b

No Gesture Detected

NO_GESTURE action fires. TTS announces "Gesture not detected. Try again!". No scores change. User can click "Play Again" to retry.

5

Result Phase

The right panel shows both moves side by side with win/loss/draw colour coding. The scoreboard updates live. The user can click Play Again (restart countdown, keep scores) or Reset Scores (return to idle, clear all data).
